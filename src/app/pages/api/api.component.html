API Documentation

Integrate temporary email functionality into your applications

Authentication

The MinuteMail API uses Bearer API key authentication for secure access to all endpoints.

API Key Authentication

  • Header: Authorization
  • Value: Bearer YOUR_API_KEY
  • Required: For all API endpoints
Authorization: Bearer your_api_key_here

Optional Mailbox Password

  • Header: X-Mailbox-Password
  • Value: Your mailbox password (if set during creation)
  • Required: Only if the mailbox was created with a password
X-Mailbox-Password: your_mailbox_password

Note: API keys are required for all endpoints and can be obtained from your MinuteMail account dashboard. The X-Mailbox-Password header is optional and only needed if you set a password when creating the mailbox.

Endpoints

POST /v1/mailbox/create

1. Create Mailbox

Headers:

Header Required Description
Authorization Yes Bearer API key for authentication
X-Mailbox-Password No Custom password for the mailbox

Request Body (JSON):

Field Type Required Description
password string No Optional mailbox password (plain text)
host string No Optional host specification
source string No Source identifier (defaults to "api")

Response (HTTP 200):

{
  "message": "Mailbox created",
  "name": "friendly-pear-shaped-cougar-191",
  "ttl": 3600
}
GET /v1/mailbox/{name}

2. Fetch All Mails

URL Path Parameters:

Name Type Description
name string Mailbox identifier (from creation)

Headers:

Header Required Description
Authorization Optional Bearer token for authenticated access
X-Mailbox-Password Optional Mailbox password (if set during creation)

Response (HTTP 200):

{
  "mails": [ /* array of MailRecord */ ],
  "expireAt": "2025-07-02T15:23:45.123456+00:00"
}
  • mails: array of all stored mail records (in insertion order)
  • expireAt: ISO-8601 timestamp when this mailbox will expire
GET /v1/mailbox/{name}/mail/{mail_id}/attachment/{filename}

3. Fetch Mail Attachment

URL Path Parameters:

Name Type Description
name string Mailbox identifier
mail_id string id field of the mail containing the attachment
filename string Name of the attachment file to download

Headers:

Header Required Description
Authorization Optional Bearer token for authenticated access
X-Mailbox-Password Optional Mailbox password (if set during creation)

Response (HTTP 200):

Returns the raw attachment file with appropriate Content-Type and Content-Disposition headers for download.

Response Headers:

  • Content-Type: MIME type of the file
  • Content-Disposition: attachment; filename="{filename}"
  • Content-Length: Size of the file in bytes
DELETE /v1/mailbox/{name}

4. Delete Entire Mailbox

URL Path Parameters:

Name Type Description
name string Mailbox identifier

Headers:

Header Required Description
Authorization Optional Bearer token for authenticated access
X-Mailbox-Password Optional Mailbox password (if set during creation)

Response (HTTP 200):

{
  "message": "Mailbox deleted"
}
DELETE /v1/mailbox/{name}/mail/{mail_id}

5. Delete Single Mail

URL Path Parameters:

Name Type Description
name string Mailbox identifier
mail_id string id field of the mail to remove

Headers:

Header Required Description
Authorization Optional Bearer token for authenticated access
X-Mailbox-Password Optional Mailbox password (if set during creation)

Response (HTTP 200):

{
  "message": "Mail deleted"
}

Usage Examples

1. Create a mailbox with password

curl -X POST https://api.minutemail.co/v1/mailbox/create \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your_api_key_here" \
  -d '{ "password": "hunter2", "source": "api" }'

2. Create a mailbox without password

curl -X POST https://api.minutemail.co/v1/mailbox/create \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your_api_key_here" \
  -d '{ "source": "api" }'

3. Fetch all mails (with mailbox password)

curl https://api.minutemail.co/v1/mailbox/q9x2k8 \
  -H "Authorization: Bearer your_api_key_here" \
  -H "X-Mailbox-Password: hunter2"

4. Fetch all mails (without mailbox password)

curl https://api.minutemail.co/v1/mailbox/q9x2k8 \
  -H "Authorization: Bearer your_api_key_here"

5. Download an attachment

curl https://api.minutemail.co/v1/mailbox/q9x2k8/mail/4f8b1a2e3c9d4f5a8b7c6d0e1f2a3b4c/attachment/document.pdf \
  -H "Authorization: Bearer your_api_key_here" \
  -H "X-Mailbox-Password: hunter2" \
  -o document.pdf

6. Delete a single mail

curl -X DELETE https://api.minutemail.co/v1/mailbox/q9x2k8/mail/4f8b1a2e3c9d4f5a8b7c6d0e1f2a3b4c \
  -H "Authorization: Bearer your_api_key_here" \
  -H "X-Mailbox-Password: hunter2"

7. Delete entire mailbox

curl -X DELETE https://api.minutemail.co/v1/mailbox/q9x2k8 \
  -H "Authorization: Bearer your_api_key_here" \
  -H "X-Mailbox-Password: hunter2"

Base URL

https://api.minutemail.co

Rate Limiting

Note: API endpoints may be subject to rate limiting. Please implement appropriate retry logic with exponential backoff in your applications.

Getting API Keys

API keys are required for all API endpoints. To obtain an API key:

  1. Sign up for a MinuteMail account
  2. Navigate to the API Keys section in your dashboard
  3. Create a new API key
  4. Use the key in the Authorization header as shown in the examples above
