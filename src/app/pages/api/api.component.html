API Documentation

Integrate temporary email functionality into your applications

Authentication

All endpoints operating on an existing mailbox support two interchangeable auth methods; choose one:

1. HTTP Basic Auth

  • Username: mailbox name
  • Password: mailbox password (if set; otherwise no auth required)
  • Header:
Authorization: Basic BASE64(<name>:<password>)

2. Custom Header

  • Header key: X-Mailbox-Password
  • Value: mailbox password (if set; omit header if none)
  • Example:
X-Mailbox-Password: hunter2

Note: The mailbox name returned by creation is both the identifier and the Basic-Auth username.

Endpoints

POST /v1/mailbox/create

1. Create Mailbox

Request Body (JSON):

Field Type Required Description
password string No Optional mailbox password (plain text)

Response (HTTP 200):

{
  "message": "Mailbox created",
  "name": "friendly-pear-shaped-cougar-191",
  "ttl": 3600
}
GET /v1/mailbox/{name}

2. Fetch All Mails

URL Path Parameters:

Name Type Description
name string Mailbox identifier (from creation)

Headers:

Use either Basic Auth or X-Mailbox-Password as described above.

Response (HTTP 200):

{
  "mails": [ /* array of MailRecord */ ],
  "expireAt": "2025-07-02T15:23:45.123456+00:00"
}
  • mails: array of all stored mail records (in insertion order)
  • expireAt: ISO-8601 timestamp when this mailbox will expire
GET /v1/mailbox/{name}/mail/{mail_id}/attachment/{filename}

3. Fetch Mail Attachment

URL Path Parameters:

Name Type Description
name string Mailbox identifier
mail_id string id field of the mail containing the attachment
filename string Name of the attachment file to download

Headers:

Use either Basic Auth or X-Mailbox-Password as described above.

Response (HTTP 200):

Returns the raw attachment file with appropriate Content-Type and Content-Disposition headers for download.

Response Headers:

  • Content-Type: MIME type of the file
  • Content-Disposition: attachment; filename="{filename}"
  • Content-Length: Size of the file in bytes
DELETE /v1/mailbox/{name}

4. Delete Entire Mailbox

URL Path Parameters:

Name Type Description
name string Mailbox identifier

Headers:

Use either Basic Auth or X-Mailbox-Password.

Response (HTTP 200):

{
  "message": "Mailbox deleted"
}
DELETE /v1/mailbox/{name}/mail/{mail_id}

5. Delete Single Mail

URL Path Parameters:

Name Type Description
name string Mailbox identifier
mail_id string id field of the mail to remove

Headers:

Use either Basic Auth or X-Mailbox-Password.

Response (HTTP 200):

{
  "message": "Mail deleted"
}

Usage Examples

1. Create a mailbox with password

curl -X POST https://api.minutemail.co/v1/mailbox/create \
  -H "Content-Type: application/json" \
  -d '{ "password": "hunter2" }'

2. Fetch all mails (Basic Auth)

curl https://api.minutemail.co/v1/mailbox/q9x2k8 \
  -H "Authorization: Basic cTl4Mms4Omh1bnRlcjI="

3. Fetch all mails (custom header)

curl https://api.minutemail.co/v1/mailbox/q9x2k8 \
  -H "X-Mailbox-Password: hunter2"

4. Download an attachment

curl https://api.minutemail.co/v1/mailbox/q9x2k8/mail/4f8b1a2e3c9d4f5a8b7c6d0e1f2a3b4c/attachment/document.pdf \
  -H "X-Mailbox-Password: hunter2" \
  -o document.pdf

5. Delete a single mail

curl -X DELETE https://api.minutemail.co/v1/mailbox/q9x2k8/mail/4f8b1a2e3c9d4f5a8b7c6d0e1f2a3b4c \
  -H "X-Mailbox-Password: hunter2"

6. Delete entire mailbox

curl -X DELETE https://api.minutemail.co/v1/mailbox/q9x2k8 \
  -H "X-Mailbox-Password: hunter2"
